# 実装計画書 — うちの子カルテ

## フェーズ分け（最小限のv1完成までの4〜5フェーズ。各フェーズの完了条件）

### フェーズ1：静的ページ骨格とデザイン土台

目的：GitHub Pagesでそのまま公開できる1ページの土台を作る。

実装範囲：

- `index.html`に、ヘッダー、ヒーロー、今日の記録、過去カレンダー、レポート出力、設定、フッターを配置する。
- `style.css`に、配色、タイポグラフィ、カード、ボタン、絵文字ボタン、レスポンシブレイアウトを定義する。
- モバイルファーストで、320px幅でも操作できる余白とボタンサイズにする。
- JavaScriptなしでもページ内容が読める最低限のHTML構造にする。

完了条件：

- ブラウザで`index.html`を直接開いて、全セクションが縦に表示される。
- スマホ幅とデスクトップ幅でテキストやボタンがはみ出さない。
- 「獣医に堂々と話せる」価値がヒーローとレポート出力セクションで伝わる。

### フェーズ2：ペットプロフィールと日次記録の保存

目的：毎日1分で記録でき、再読み込み後もlocalStorageから復元される状態にする。

実装範囲：

- `script.js`にlocalStorage操作関数を作る。
- 初回起動時にペットプロフィールを作成できるようにする。
- 今日の日付の記録フォームを表示し、食事量、排便、気分、気になること、メモを保存する。
- 同じ日の記録は再編集できるようにする。
- 保存完了、未保存、入力不足の状態表示を作る。

完了条件：

- ペット名を登録すると、再読み込み後も同じペットが表示される。
- 今日の記録を保存すると、localStorageに`uchinoko:pets/{id}/records/{date}`形式で保存される。
- 保存済みの記録を編集して再保存できる。
- 外部API通信が発生しない。

### フェーズ3：過去カレンダーと「いつもと違う」検出

目的：記録の羅列ではなく、受診時に伝えるべき変化を発見できる画面にする。

実装範囲：

- 直近14日のカレンダーまたは日付リストを表示する。
- 記録済み、未記録、注意、いつもと違うの状態を視覚的に分ける。
- 食事、排便、気分、気になることのルールベース検出を実装する。
- 日付を押すと、その日の詳細と検出理由が表示される。
- 未入力日は異常扱いにしない。

完了条件：

- `half`、`none`、`diarrhea`、`weak`などの記録に赤いハイライトが付く。
- `less`が2日連続した場合など、連続変化の検出が動く。
- 検出理由が「食事が2日連続で少なめ」のように人間が読める文で表示される。
- カレンダーから日別詳細を確認できる。

### フェーズ4：受診用レポートと印刷ビュー

目的：獣医に見せるための過去2週間レポートを作り、PDF保存できる状態にする。

実装範囲：

- 対象ペットの過去14日分を集計する`buildVetReport()`を実装する。
- 食事低下日数、排便異常日数、気分低下日数、気になることの頻度、最初の変化日をサマリー化する。
- 画面内にレポートプレビューを表示する。
- 印刷専用レイアウトをCSSの`@media print`で定義する。
- 「印刷ビューを開く」から`window.print()`を呼び出す。

完了条件：

- 過去2週間のサマリーが日次記録から生成される。
- 赤ハイライトの理由が印刷ビューにも残る。
- A4縦で1〜2ページに収まる。
- ブラウザの印刷機能からPDF保存できる。

### フェーズ5：仕上げ、アクセシビリティ、応募準備

目的：コンテスト応募作として見せられる完成度まで整える。

実装範囲：

- OGP画像、favicon、meta descriptionを設定する。
- ボタンに適切な`aria-pressed`、フォームに`label`を設定する。
- キーボード操作で主要入力と印刷ビュー起動ができるようにする。
- データ削除前の確認ダイアログを実装する。
- READMEまたは応募文に、個人情報外部送信ゼロと医療判断をしない注記を書く。

完了条件：

- 手動テストチェックリストがすべて通る。
- Lighthouseのアクセシビリティで大きな警告がない。
- GitHub Pagesの公開URLで、スマホとPCの両方から操作できる。
- 応募締切の2026-06-30までにURL、説明文、スクリーンショットがそろう。
